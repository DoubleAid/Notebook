# 交通标识牌识别

核心：目标检测 + 分类，两步完成：先定位标识牌，再识别类别

交通标识牌的核心诉求：

小目标精准检测（标识牌尺寸小）+ 多类别分类（限速 / 让行 / 禁止等）+ 抗远距离 / 模糊，

分「检测 + 分类一体化模型」（首选）和「检测 + 分类两步模型」（灵活适配），按落地优先级排序：

## 【工业界首选｜检测 + 分类一体化｜端到端最优】

### YOLO 系列（YOLOv5/v8/v9/v10，强推！）***

自动驾驶感知第一首选模型，你的场景必用 YOLO，没有之一！

+ 核心优势：检测 + 分类一步完成，端到端输出标识牌的位置框 + 类别（如限速 60 / 禁止左转 / 直行），速度 + 精度双天花板，支持小目标检测（远距离标识牌），抗模糊 / 光照干扰强。
+ 选型建议（按落地优先级）：
    + YOLOv8n/s/m：车载端侧首选，v8n 轻量化版本端侧帧率 80+FPS，v8m 精度足够，完美适配标识牌检测；
    + YOLOv9/YOLOv10：2024-2025 最新版本，在小目标检测 / 推理速度上再升级，新项目优先用 v10；
+ 适配场景：所有交通标识牌（限速、警告、禁令、指示类），远距离小标识牌 / 遮挡标识牌 / 模糊标识牌均可精准识别；
+ 工程优势：代码开源成熟、支持 PyTorch/ONNX/TensorRT 部署、易与 SLAM 融合（标识牌坐标可作为视觉地标，辅助 SLAM 回环检测 / 重定位）；
+ 关键优化：对标识牌小目标做马赛克增强 / 多尺度训练，检测精度可提升 10%+。

## 【速度优先｜极致轻量化端侧部署】（算力极低：如车载 MCU / 低端 NPU）

### YOLOv8n/YOLOv10n + 模型量化 / 剪枝

+ 核心操作：对 YOLOv8n/v10n 做INT8 量化 / 通道剪枝，参数量压缩至 1M 内，推理帧率达 100+FPS，精度仅下降 3%-5%，车载最低算力场景必选。

### MobileNet-SSD / NanoDet（超轻量检测）

+ 核心优势：MobileNet-SSD 是经典轻量化检测模型，NanoDet 是无锚框轻量检测，参数量仅几百 K，端侧推理无压力；
+ 适配场景：低速自动驾驶（园区车 / 环卫车）、近距离标识牌识别，对精度要求不高的场景。

## 【高精度｜科研 / 复杂场景】（如远距离极小标识牌 / 恶劣天气）

### Faster R-CNN + ResNet50/101 + FPN

+ 核心优势：两阶段检测经典 SOTA，FPN 特征金字塔对小目标标识牌检测精度拉满，适合极端场景（如高速远距离极小限速牌、大雾天模糊标识牌）；
+ 注意点：推理速度慢，适合科研 / 离线分析，量产需做模型加速（TensorRT 优化）。

### DETR/YOLO-World（Transformer 检测）

+ 核心优势：DETR 是纯 Transformer 检测模型，无锚框设计，解决标识牌漏检 / 误检问题；YOLO-World 支持零样本 / 少样本分类，对罕见标识牌（非标标识）识别能力强；
+ 适配场景：科研高精度场景、需要识别罕见交通标识牌的项目。
