# 如何规避GNSS误差影响多传感器融合

## 源头防御

+ 使用多频多系统接收机
+ 安装高精度天线和接收器

## 过程抑制

+ 采用更鲁棒的融合架构：紧组合与深组合
  + 松组合：直接使用GNSS的PVT解。GNSS内部定位引擎的任何故障都会将错误结果直接注入滤波器，难以检测和隔离。
  + 紧组合的优势：将原始伪距/载波相位观测值直接输入滤波器，与IMU、轮速计等其他传感器状态一起估计。

+ 设计鲁棒的状态估计算法与残差处理
  + 自适应卡尔曼滤波：动态调整观测噪声协方差矩阵R。当检测到GNSS观测质量下降（如城市峡谷）时，自动增大R，降低滤波器对GNSS观测的信任权重，更多地依赖IMU短期精度。
  + 鲁棒损失函数/鲁棒核函数：在优化框架（如Ceres, g2o）中，对残差使用Huber、Cauchy等鲁棒核函数，降低异常大残差对整体代价函数的影响

+ 严密的完好性监测与故障检测隔离
  + 传感器一致性检查：比较GNSS速度与IMU/轮速计推算的速度，比较GNSS航向与IMU/电子罗盘给出的航向，出现显著不一致时对GNSS数据报警
